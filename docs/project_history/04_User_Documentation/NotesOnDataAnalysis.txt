多模穴多維度SPC統計管制策略
針對多模穴（Multi-cavity）模具且單一產品上有多個尺寸監測點（例如座標 X/Y、多個關鍵尺寸），導致數據量龐大且製程表現不一致的情況，AIAG-VDA SPC 手冊建議採用**多變量/多維度（Multivariate/Multidimensional）**的統計方法與特定的管制圖策略來處理。
以下是針對此複雜情況的具體處理建議：
1. 採用多變量管制圖 (Multivariate Control Charts)
當一個零件上有多個監測點（例如位置度的 X 軸與 Y 軸座標），且這些特性之間可能存在交互影響時，單獨監控每一個特性（如分別畫 X 的管制圖和 Y 的管制圖）可能會誤判。
• 適用工具：手冊推薦使用 Hotelling’s T 
2
  管制圖 或 多變量指數加權移動平均管制圖 (MEWMA) [1]。
• 運作原理：這些圖表將多個相關的特性（如多個監測點的數據）結合成單一的統計量進行監控。
• 優點：能夠偵測到單一變數圖表無法發現的聯合偏移（例如 X 和 Y 都還在界限內，但組合起來的位置度已經超標）。
• 缺點：一旦發生失控訊號，較難直接識別具體是哪一個特性（或哪一個監測點）導致了不穩定，需要進一步分析 [1]。
2. 針對幾何座標與位置度的處理 (GD&T / Coordinates)
對於多點監測中常見的幾何尺寸與公差（GD&T），特別是涉及 2D 或 3D 座標（如位置度）的情況：
• 多維常態分佈模型：手冊建議使用多變量常態分佈模型，並利用**超橢球體（Hyper-ellipsoid）**來定義隨機變異範圍，而非單純的線性範圍 [2, 3]。
• 組裝間隙法 (Assembly Clearance)： 針對需配合的特性（如孔徑大小與位置度），可以計算「實際組裝間隙 C」。 公式概念為：C=
text尺寸變數+
text位置變數−
text最大實體虛擬尺寸。 透過分析 C 的分佈（如計算 $C\_{50%}$ 與 $C\_{0.135%}$），並要求 C
ge0，可以將多個監測點的複雜問題轉化為單一指標來評估能力 [4, 5]。
3. 解決製程表現不一致的策略
當各監測點或各模穴的表現不一致（Inconsistent）時：
• 多模穴/多主軸的獨立性評估： 如果每個模穴或監測點的重複性條件不同，原則上應將其視為獨立的機器處理 [6]。但若數量龐大（如 32 穴 x 5 個測點），這在經濟上不可行。
• 簡化驗收程序 (幾何均一性)： 在量產前，必須先證明所有模穴在統計上是「幾何均一」的（Geometrically identical）。這意味著模穴 A 的測點 1 與模穴 B 的測點 1 必須具有可比較性。若證實均一，則可混合評估；若不均一，則需針對表現最差的模穴或特定測點進行個別改善 [7]。
• 標準化管制圖 (Z-Chart / Short Run SPC)： 若要在一張圖上監控多個表現不一的特性（例如測點 A 變異小，測點 B 變異大），可以使用 Z-Chart。透過將數據標準化（減去平均值並除以標準差），可以將不同單位的測點數據繪製在同一張圖上，以監控整體的穩定性 [8, 9]。
4. 抽樣與報告策略
• 抽樣：對於這種複雜製程（通常屬於模型 C 或 D），建議採用小樣本但高頻率的抽樣策略，或者採用輪替抽樣（每次抽不同模穴/測點），以確保能捕捉到系統性的變化 [10, 11]。
• 標籤追溯：必須對每個測量值進行標籤化（Labeling），註明其來自哪個模穴、哪個主軸以及哪個測點位置。這樣當多變量管制圖發出警報時，才能回溯找出具體的變異來源 [12, 13]。
總結：組裝間隙法。若需個別監控多個特性，則建議使用 Z-Chart 進行標準化管理，並確保有完善的數據追溯系統以識別特定模穴的偏差。

計量型管制圖定義與多模穴應用指南
根據 AIAG-VDA SPC 手冊，計量型管制圖（Variable Control Charts）是用於監控連續性測量數據（如長度、壓力、時間）的工具。以下針對您的提問，分為「管制圖種類與定義」、「使用與解讀方式」以及「多模穴模具的對應策略」三個部分進行說明。
1. 常見的計量型管制圖種類
計量型管制圖通常由兩張圖組成：一張監控製程位置（Location/Center），另一張監控製程變異（Variation/Spread）。
• 平均值與標準差管制圖 (
barx-s Chart)
    ◦ 定義： 使用子組平均值 (
barx) 監控位置，使用子組標準差 (s) 監控變異。
    ◦ 適用時機： 這是電腦輔助計算時的首選圖表。適用於子組樣本數較大（通常 n
ge5）的情況，因為標準差比全距（Range）更能有效估計變異 [1], [2]。
• 平均值與全距管制圖 (
barx-R Chart)
    ◦ 定義： 使用子組平均值 (
barx) 監控位置，使用子組全距 (R) 監控變異。
    ◦ 適用時機： 適用於手動計算且樣本數較小（通常 $n < 10$）的情況 [3], [4]。
• 中位數與全距管制圖 (
tildex-R Chart)
    ◦ 定義： 使用子組中位數 (
tildex) 監控位置，使用全距 (R) 監控變異。
    ◦ 適用時機： 適合手動繪製（計算簡單）或自動化測量中數據變異極大的情況。中位數受極端值（離群值）影響較小，但對製程變化的反應較慢 [3], [5]。
• 單值移動全距管制圖 (I-MR / X-MR Chart)
    ◦ 定義： 使用單個測量值 (x) 監控位置，使用相鄰兩個數值的移動全距 (MR) 監控變異。
    ◦ 適用時機： 當無法進行分組抽樣時（例如：破壞性測試、檢測成本極高、或是均質性極高的液體/粉末製程，n=1）[3], [6]。

2. 定義、使用與解讀原則
定義與核心概念
• 管制界限 vs. 規格界限： 管制圖上的「管制界限（Control Limits）」是基於製程統計數據計算出來的（通常是平均值
pm3
sigma），用來區分隨機原因與特殊原因。它決不能與圖紙上的「規格界限（Tolerances/Specification Limits）」混淆。現場操作用的管制圖通常不顯示規格界限，以免誤導 [7], [8]。
• 製程導向 vs. 公差導向：
    ◦ 製程導向 (Shewhart)： 用於優化品質與持續改進，目標是零缺陷。界限基於製程自身變異 [8]。
    ◦ 公差導向 (Acceptance)： 用於確保產品符合公差，允許一定比例的不良品。界限基於公差計算 [9]。
使用流程
1. 收集數據： 依據抽樣策略（如每小時抽 5 件）收集數據。
2. 計算界限： 利用公式計算 UCL（上管制界限）與 LCL（下管制界限）。
3. 描點監控： 將數據點繪製於圖上。
4. 即時反應： 若發現異常，需依據「失控反應計畫（OCAP）」採取行動 [10]。
解讀判斷（穩定性準則）
依據標準規則（如 Western Electric Rules），以下情況視為製程不穩定（失控），需調查特殊原因：
• 超出界限： 任何一點超出
pm3s 管制界限 [11]。
• 連串 (Runs)： 連續 7 點或 9 點落在中心線的同一側 [12]。
• 趨勢 (Trends)： 連續 7 點呈現持續上升或下降 [12]。
• 區域規則： 例如連續 3 點中有 2 點超出
pm2s 警告界限 [11]。

3. 多模穴（Multi-Cavity）特性的對應使用策略
對於具備多模穴（Multi-cavity）、多主軸或多夾具的設備，每個模穴或工位在統計上可能具有不同的製程位置（平均值）或變異，因此被視為複雜製程。
挑戰
若將所有模穴的數據混合繪製在同一張標準
barx-R 圖上，組內變異（模穴間的差異）可能會掩蓋掉製程隨時間的變化，或者導致管制界限過寬而失去敏感度 [13]。
對應策略
根據 AIAG-VDA 手冊建議，可採用以下方式處理：
1. **視為獨立機器（最嚴謹但成本高）：**將每個模穴視為獨立的生產單元，分別進行能力研究與管制。這能最精確地控制，但在模穴數眾多時（如 32 穴）極不經濟 [13]。
2. 分組抽樣與幾何均一性確認（簡化法）：
• 在驗收階段，需證明不同模穴（例如模穴 1 與模穴 2）在統計上是「幾何均一」的（Geometrically identical）。
• 若均一性成立，可將所有測量結果視為一個整體分佈進行評估。若發現異常值，再透過標籤追溯回特定模穴 [14], [15]。
3. **使用特定的管制圖模型（Model C 類別）：**多模穴製程通常符合 ISO 22514-2 定義的 Model C（位置隨模穴不同而改變，但變異通常是固定的常數）[16]。
• 擴展的 Shewhart 管制圖 (Extended Shewhart Chart)： 這種圖表的管制界限計算中，額外包含了一個項來容許模穴之間的平均值差異（Systematic differences）。這樣可以避免因為模穴間固有的偏差而頻繁誤報失控 [17], [18]。
• 允收管制圖 (Acceptance Control Chart)： 若製程能力極高（變異遠小於公差），且允許模穴平均值在一定範圍內偏移，可使用此圖。它是基於公差設定界限，而非製程平均值 [19], [18]。
4. 抽樣建議：小樣本數但較高頻率的抽樣策略，或者採用輪替抽樣（每次抽不同模穴），以監控整體系統的穩定性 [18]。

不穩定製程的統計管制圖選用指南
針對不穩定製程（即製程的位置或變異隨時間變化，未處於統計受控狀態），標準的 Shewhart 管制圖可能會頻繁發出「錯誤警報」。根據 AIAG-VDA SPC 手冊，選用合適的管制圖需依據**時間依賴分佈模型（Time-Dependent Distribution Models）**來決定。
這些不穩定製程通常被歸類為模型 B、C 或 D（根據 ISO 22514-2）。以下是針對不同不穩定類型的選用建議：
1. 針對「位置不穩定」的製程 (Model C)
這類製程的變異（標準差）是穩定的，但平均值（位置）會隨時間發生系統性或隨機的變化。最典型的例子是**刀具磨耗（Tool Wear）**造成的趨勢，或批次更換導致的跳動。
• 推薦圖表 1：允收管制圖 (Acceptance Control Chart) [1]
    ◦ 適用情境：當製程位置的變化是系統性的且被允許的（例如刀具磨耗），且目前的製程變異遠小於公差（通常要求
hatsigma
leT/10）[2]。
    ◦ 特點：這是一種「公差導向」的管制圖。管制界限是基於公差和製程變異計算的，允許平均值在一定範圍內漂移 [2]。
    ◦ 注意：此圖表無法實現零缺陷策略，因為它允許一定比例的不良品存在 [3]。
• 推薦圖表 2：擴展 Shewhart 管制圖 (Extended Shewhart Chart) [1, 4]
    ◦ 適用情境：當預期會有製程固有的平均值變化（如趨勢、環境影響、不同設備生產）時使用。
    ◦ 特點：這是一種「製程導向」的管制圖。在計算管制界限時，會額外增加一個項（term）來包含平均值的波動範圍 [4]。
    ◦ 優勢：相較於標準 Shewhart 圖，它擴大了管制界限，避免因已知的平均值漂移而頻繁報警。
2. 針對「變異不穩定」的製程 (Model B)
這類製程的位置（平均值）是固定的，但變異（標準差）會隨時間改變。這在控制上較為困難。
• 推薦圖表：Shewhart 管制圖 (但需調整抽樣策略) [1]
    ◦ 雖然建議使用標準 Shewhart 圖（如 S 圖或 R 圖），但由於變異不穩定，很難重新調整。
    ◦ 關鍵策略：通常建議使用較大的樣本數 (Larger Sample Size)，以便更準確地捕捉變異的變化 [1, 5]。如果變異變化很大，可能需要針對不同的變異狀態分別設定管制界限，或者致力於消除導致變異不穩定的根本原因。
3. 針對「位置與變異皆不穩定」的製程 (Model D)
這是風險最高的情況，位置和變異都隨時間發生系統性或隨機的變化。
• 推薦圖表：允收管制圖 (Acceptance Control Chart) [1]
• 替代方案：擴展 Shewhart 管制圖 [4]
    ◦ 關鍵策略：對於此類製程，手冊強烈建議同時採用較大的樣本數以及較高的抽樣頻率，以確保能及時檢測到製程的劇烈變化 [1, 6]。

從檢測到預防：SPC 製程控制轉型指南
根據 AIAG-VDA SPC 手冊，透過統計製程控制（SPC）從「檢測（Detection）」轉向「預防（Prevention）」的核心在於改變對品質管理的根本態度與操作方法。
以下是實現此轉變的關鍵機制與步驟：
1. 改變核心策略：由「反應」轉為「主動」
• 過去的模式（檢測/反應）： 傳統製造依賴生產部門製造產品，再由品質部門檢查最終產品，剔除不符合規格的產品。這種「反應式檢測（Reactive Detection）」是浪費的，因為時間和材料已經投入到無法使用的產品中，僅關注於在缺陷發生後進行識別和修復 [1]。
• SPC 的模式（預防/主動）： 為了減少浪費，必須轉向「主動預防（Proactive Prevention）」的策略。這需要利用 SPC 的要素來主動監控並及早識別問題，以便在產生缺陷之前就採取及時的糾正措施 [2]。這體現了「第一次就把事情做對（Do it right the first time）」的零缺陷策略 [2]。
**2. 將行動焦點從「產出」轉移到「製程」**SPC 強調透過「製程控制系統」來運作，其關鍵在於採取行動的對象：
• 避免僅對產出採取行動 (Action on the Output)： 僅檢測並糾正超出規格的產品（如全檢、報廢、返工）是最不經濟的，這無法解決根本的製程問題。對產出的行動僅應作為製程不穩定或能力不足時的臨時措施 [3]。
• 專注對製程採取行動 (Action on the Process)： 這是最經濟的方法。透過監控製程特性（如溫度、循環時間、進料調整等）與內部變異，在製程或產出偏離目標值過遠之前就進行調整（如操作員培訓、設備調整、材料變更）[4-6]。手冊明確指出：「重點應放在預防，而非檢測」[7]。
3. 利用統計訊號提前預警SPC 的目的不是單純為了符合規格，而是為了具成本效益地確保品質：
• 區分變異類型： SPC 定義了製程的「共同原因（隨機原因）」變異範圍。當「特殊原因（可歸屬原因）」介入導致不穩定時，管制圖會發出訊號 [8]。
• 在問題發生前行動： SPC 被視為一種預防活動，因為它能識別特殊原因的發生，讓人員能在產生不良品之前，識別來源並採取反應措施 [8, 9]。
4. 選擇「製程導向」而非「公差導向」的控制概念在選擇管制圖時，概念的選擇決定了是預防還是檢測：
• 製程導向管制圖 (Process-related Control Charts)： 這是基於預防策略。其管制界限是根據製程自身的參數（水準/變異）計算，與公差無關。其目的是優化品質、實現零缺陷生產，並持續改進製程 [10, 11]。
• 公差導向管制圖 (Tolerance-related Control Charts)： 這是基於檢測策略。其目的是確保產品符合公差（允許一定比例的不良品），而非改進製程 [11, 12]。
總結來說，從檢測轉向預防的關鍵在於： 不再依賴檢查最終產品來篩選好壞（品質控制 QC），而是透過即時監控製程參數與變異，在製程顯示出不穩定跡象但尚未產生大量不良品時，就主動介入調整製程（品質保證 QA）[7, 13, 14]。

製程性能指標 Pp 與 Ppk 的應用準則
根據 AIAG-VDA SPC 手冊與 ISO 標準（如 ISO 22514 系列），必須使用製程性能指標 (P_p,P_pk) 而非製程能力指標 (C_p,C_pk) 的主要情況，取決於製程的**穩定性（Stability）**狀態。
具體必須使用 P_p 的情況如下：
**1. 製程處於不穩定狀態 (Unstable / Out of Control)**當已知製程不符合穩定性要求，即處於「失控」狀態時，必須使用 P_p [1]。如果在評估過程中發現製程顯式地不穩定（例如管制圖出現超出界限或非隨機模式），則應使用性能指標進行評估 [2]。在製程評估矩陣中，處於不穩定狀態的區域（第二和第四象限）均使用性能指標 [3]。
**2. 穩定性尚未被調查或證實 (Stability Not Investigated/Proven)**當尚未對製程穩定性進行調查，或者目前的數據不足以證明製程處於統計受控狀態時，應使用 P_p [1]。只要穩定性未被證實，C_p 指標就不能使用 [2]。
**3. 初步製程研究階段 (Preliminary Process Performance)**在生產製程的放行階段（Release of manufacturing processes），通常進行的是「初步製程性能」研究。此時雖然使用了較大的樣本（如 n
ge125），但製程可能尚未完全暴露於所有長期的變異來源（5M：人、機、料、法、環）之下，或者僅能進行有限度的變異觀察，因此計算的是初步性能指標 P_p/P_pk [4-6]。
**4. 數據無法用於評估穩定性 (Data Limitations)**當所收集的數據類型或結構不允許進行穩定性評估時（例如缺乏時間序列資訊），應使用 P_p [1]。
**5. 僅需性能指標時 (Explicit Requirement)**如果明確要求提供性能指標，則隱含表示不需要或尚未要求提供穩定性證明，此時適用 P_p [7]。
總結區分原則：C_p/C_pk 指標僅在製程已證實滿足穩定性要求（處於統計受控或受控穩定狀態）時才能使用 [1, 2]。在所有其他情況下（不穩定、未驗證、初步階段或數據受限），都必須使用 P_p/P_pk。

AIAG-VDA 製程性能與能力計算指南
根據 AIAG-VDA SPC 手冊（基於 ISO 22514 和 ISO 3534 系列標準），製程性能 (P_p,P_pk) 與製程能力 (C_p,C_pk) 的計算公式在本質上是相同的。兩者的區別不在於數學公式，而在於數據收集的邊界條件以及製程是否已被證實處於穩定狀態[1, 2]。
手冊主要推薦使用一般幾何法（General Geometric Method），因為它適用於任何分佈類型（常態或非常態）。
以下是詳細的計算公式說明：
1. 通用計算公式（一般幾何法 / 分位數法）
此方法使用統計分佈的**分位數（Quantiles）**來計算，適用於常態分佈與非常態分佈。無論是指標 P_m,P_p 還是 C_p，皆適用以下邏輯[2, 3]：
• $X\_{50%}$：中位數（分佈的中心）。
• $X\_{0.135%}$：0.135% 分位數（相當於常態分佈的 −3
sigma 位置）。
• $X\_{99.865%}$：99.865% 分位數（相當於常態分佈的 +3
sigma 位置）。
• U：規格上限 (Upper Specification Limit)。
• L：規格下限 (Lower Specification Limit)。
A. 潛在能力/性能指標 (C_p,P_p,P_m)
衡量製程變異寬度是否小於規格公差範圍（不考慮位置偏移）。
$$ C\_p = P\_p = P\_m = \\frac{\\text{Tolerance}}{\\text{Process Spread}} = \\frac{U - L}{X\_{99.865%} - X\_{0.135%}} $$ [4]
B. 關鍵能力/性能指標 (C_pk,P_pk,P_mk)

• 上限制標 (Upper):$$ C\_{pkU} = P\_{pkU} = \\frac{U - X\_{50%}}{X\_{99.865%} - X\_{50%}} $$
• 下限制標 (Lower):$$ C\_{pkL} = P\_{pkL} = \\frac{X\_{50%} - L}{X\_{50%} - X\_{0.135%}} $$

統計製程管制：隨機與特殊變異判別準則
透過管制圖區分**隨機原因（Random Causes）與特殊原因（Special Causes）**是統計製程管制（SPC）的核心目的。Shewhart 管制圖的設計初衷正是為了區分這兩類變異來源 [1]。
根據來源文件，具體的區分邏輯與判斷準則如下：
1. 基本概念區分
• 隨機原因 (Random Causes)：
    ◦ 這是製程中固有的、隨著時間持續存在的變異來源（又稱共同原因 Common Causes）[2]。
    ◦ 當製程僅受隨機原因影響時，製程處於「統計受控」狀態，其變異是已知且可預測的（通常呈現常態分佈且落在特定範圍內）[3]。
• 特殊原因 (Special Causes)：
    ◦ 這不是製程固有的變異，而是由特定的、非經常性的因素引起的（又稱可歸屬原因 Assignable Causes）[4]。
    ◦ 當特殊原因介入時，會導致製程不穩定，需採取具體措施來消除 [5]。
2. 透過「管制界限」區分
最基本的判斷依據是計算出的管制界限（Control Limits）（通常為平均值
pm3s）：
• 隨機原因：測量點隨機散佈在管制界限之內，且無明顯規律。管制界限即是用來描述製程不可避免的隨機變異範圍 [6]。
• 特殊原因：當有測量點超出管制界限時，通常假設這是由特殊原因造成的，必須進行調查與排除 [6], [7]。
3. 透過「非隨機模式」區分 (穩定性準則)
即使所有點都在管制界限內，若數據呈現特定的非隨機排列模式，也代表特殊原因的存在。以下是常見的判定準則（Stability Criteria）：
• 連串 (Runs)：連續一定數量的點（通常 n
ge7 或 9）落在中心線的同一側 [8], [7]。這顯示製程平均值可能發生了偏移。
• 趨勢 (Trends)：連續一定數量的點（通常 n
ge7）呈現持續上升或下降 [8]。這可能暗示工具磨損或溫度變化等漸進式影響。
• 中間三分之一 (Middle Third)：落在管制圖中間三分之一區域內的點數異常多或異常少（例如：在 25 點中少於 11 點或多於 23 點），這可能暗示數據分層或過度控制 [9]。
• 區域法則 (Zone Rules / Western Electric Rules)：針對常態分佈製程的進階規則，若滿足以下任一條件即視為特殊原因 [7]：
    ◦ 連續 3 點中有 2 點 落在同一側且超出
pm2s 界限。
    ◦ 連續 5 點中有 4 點 落在同一側且超出
pm1s 界限。
    ◦ 連續 15 點 全部落在中心線
pm1s 範圍內（顯示變異過小，可能是分層取樣問題）。
總結
在管制圖上，凡是超出管制界限的點或符合非隨機模式（如連串、趨勢）的點，應被視為由特殊原因引起，代表製程不穩定；反之，若點隨機分佈於界限內，則視為僅受隨機原因影響，製程處於受控狀態。

AIAG-VDA SPC 離群值處理準則
根據「AIAG-VDA SPC Manual」手冊，處理管制圖中離群值（Outliers）的正確方法包含定義識別、處置方式以及根本原因調查。
以下是詳細的處理步驟與原則：
1. 正確定義離群值首先需釐清離群值的定義。離群值並非指超出公差（Tolerance）的數值。
• 定義：離群值是指**不屬於被調查母體（Population）**的數值。這意味著該數值並非由被調查的製程所產生，因此無法代表該製程 [1], [2]。
• 常見原因：離群值通常源自於量測錯誤、重複量測同一工件、數據輸入或傳輸錯誤、混料、操作錯誤或不合適的量測程序 [2]。此外，若研究目的是評估量產能力，則「校準」或「設置（Setup）」過程的量測值也應視為離群值 [2], [3]。
**2. 識別離群值 (Identification)**單靠統計檢定不足以判定離群值。
• 統計限制：離群值檢定通常只能判斷數值是否符合假設的分佈模型（如常態分佈），因此僅能提供「疑似」離群值的跡象，無法直接認定 [4]。
• 必要條件：將一個數值認定為離群值，必須有明確的理由（Clear justification），證明該量測值不可能來自原本的製程 [4]。必須記錄離群值的發生並加以識別 [4]。
**3. 處置方式 (Handling)**一旦確認為離群值，應採取以下具體行動：
• 不可刪除：離群值不應從數據集中被刪除（deleted），因為這可能導致資訊遺失 [5]。
• 標記為無效：應將離群值保留在數據集中，但在評估時將其標記為無效（invalid），以確保其被「忽略」 [5]。
• 排除於計算之外：已被識別為離群值的量測數據，不得包含在統計參數的計算中（如變異範圍、製程位置、能力指標等），否則會導致評估結果失真 [3], [5]。
4. 糾正措施
• 調查原因：離群值代表一種錯誤，可能導致損失。因此，必須調查造成離群值的根本原因 [5]。
• 消除原因：必須採取措施消除造成離群值的原因，以防止再次發生 [5]。
總結來說，處理離群值的關鍵在於**「記錄但不計算」**，並確保每一個被排除的數據都有明確的技術理由，而非僅因數值過大或過小而排除。

AIAG-VDA SPC 管制圖選擇與應用指南
根據「AIAG-VDA SPC Manual」來源文件，選擇最合適的管制圖需依據數據類型、樣本大小、製程分佈特性（常態或非常態）、以及製程隨時間的行為模式（如是否有趨勢或工具磨耗）來決定。
以下是根據不同製程特性選擇管制圖的詳細指南：
1. 依據數據類型選擇 (Characteristic Type)
首先需區分數據是計量型（Variable/Continuous）還是計數型（Attribute/Discrete） [1], [2]。
A. 計量型數據 (Variable Data)
這類數據來自測量（如長度、壓力），通常能提供更多製程資訊。選擇依據樣本數（Subgroup Size, n）：
• 單個測量值 (n = 1)：
    ◦ 適用於破壞性測試、檢測成本高昂、或製程非常穩定且均質（如化學液體）的情況 [3]。
    ◦ 推薦圖表： 單值移動全距管制圖 (I-MR chart / X-MR chart) [4], [5]。
• 小樣本 (1 < n < 10)：
    ◦ 適用於手動計算或傳統抽樣。
    ◦ 推薦圖表： 平均值與全距管制圖 (X-bar & R chart) [4], [6]。
    ◦ 備註：若為自動化測量且數據變異大，可考慮中位數圖 (Median chart)，但其反應較慢 [7]。
• 較大樣本 (n ≥ 10) 或使用電腦計算：
    ◦ 當樣本數增加時，全距（Range）估計標準差的效率會降低，因此應改用標準差。
    ◦ 推薦圖表： 平均值與標準差管制圖 (X-bar & s chart)。這是電腦輔助計算時的推薦標準圖表 [8], [4], [9]。
B. 計數型數據 (Attribute Data)
這類數據來自計數（如不良品數量、缺陷數）。選擇依據是「計件（不良品）」還是「計點（缺陷數）」，以及樣本大小是否固定 [4]。
• 計件 (Defectives - 不良品數量/比例)：
    ◦ 樣本數固定：使用 np管制圖 (np-chart) [4], [10]。
    ◦ 樣本數不固定：使用 p管制圖 (p-chart) [4], [11]。
• 計點 (Defects - 缺陷數量)：
    ◦ 樣本單位/大小固定：使用 c管制圖 (c-chart) [4], [12]。
    ◦ 樣本單位/大小不固定：使用 u管制圖 (u-chart) [4], [13]。

--------------------------------------------------------------------------------
3. 依據檢測靈敏度需求選擇 (Detection Sensitivity)
如果製程需要檢測非常微小的偏移（Small shifts），傳統 Shewhart 管制圖可能反應不夠快。
• 檢測微小偏移：
    ◦ CUSUM (累積和管制圖)： 對小幅度偏移非常敏感，能快速顯示趨勢變化 [18]。
    ◦ EWMA (指數加權移動平均管制圖)： 利用歷史數據的加權平均來檢測小偏移，適合用於漸進式變化的監控 [19]。
4. 其他特殊製程情境
• 小批量/多樣少量 (Short Runs/High Mix)： 使用 Z-Chart 或 Short Run SPC 技術，將不同產品的數據標準化後繪製在同一張圖上 [4], [20]。
• 多變量特性 (Multivariate)： 當多個特性相互影響（如位置度座標 X 和 Y）時，應使用 Hotelling’s T² chart 或 MEWMA，而非單獨監控各個特性 [21]。
總結選擇邏輯
在選擇時，請優先考慮以下順序：
1. 數據是測量的（計量）還是計數的（計數）？ (計量型優於計數型，因為更適合零缺陷策略 [22], [23])。
2. 樣本數大小為何？ (決定用 R 還是 s，或是 I-MR)。
3. 製程是否為常態分佈？ (若否，考慮 Pearson 或數據轉換)。
4. 是否有已知趨勢（如磨耗）？ (若有，考慮允收管制圖或擴展 Shewhart)。
5. 是否需要檢測微小變化？ (若需要，考慮 CUSUM 或 EWMA)。

統計製程穩定性判斷準則指南
判斷製程是否處於受控狀態（In Control / Statistically Stable），主要依賴**管制圖（Control Charts）**的分析，透過統計方法區分製程變異是來自「隨機原因（Random Causes）」還是「特殊原因（Special Causes）」[1-3]。
根據來源文件，判斷製程穩定性（Stability）的具體標準與方法如下：
1. 基本判斷邏輯
• 隨機原因 vs. 特殊原因：一個受控的製程，其變異僅由隨機原因（固有變異）造成，且變異模式是已知且可預測的[1]。若觀測到非隨機的行為，則視為違反穩定性[4]。
• 管制界限（Control Limits）：判斷的依據是計算出的「管制界限」，而非產品的「規格界限（Tolerances）」。管制圖不應包含規格界限，以免混淆[3]。
**2. 具體判定準則 (Stability Criteria)**通常使用以下統計準則來識別製程的不穩定性（Out-of-Control）：
• 超出管制界限：任何一點超出上管制界限（UCL）或下管制界限（LCL）（通常為
pm3s）[4, 5]。
• 連串（Runs）：連續一定數量的點（通常 n
ge7 或 9）落在中心線的同一側[5, 6]。
• 趨勢（Trends）：連續一定數量的點（通常 n
ge7）呈現持續上升或下降的走勢[6]。
• 中間三分之一（Middle Third）：落在管制界限中間三分之一區域內的點數異常多或異常少（例如：在 25 個樣本點中，少於 11 個或多於 23 個點落在此區間，這可能暗示數據分層或過度控制）[6, 7]。
**3. 進階判定規則 (Western Electric / Nelson Rules)**針對常態分佈的製程，文件列出了基於標準差（s 或
sigma）的具體判定規則（如下圖所示，違反任一條即視為不穩定）[5]：
1. 1 點超出
pm3s 界限。
2. 連續 3 點中有 2 點落在同一側且超出
pm2s 界限。
3. 連續 5 點中有 4 點落在同一側且超出
pm1s 界限。
4. 連續 9 點落在中心線的同一側。
5. 連續 15 點落在中心線
pm1s 範圍內（顯示變異過小，可能是數據分層或量測問題）。
4. 不同階段的評估方式
• 機器性能研究階段 (Machine Performance)：由於樣本數較少（如 50 件），通常進行定性穩定性評估。需觀察數值曲線是否有無法解釋的離群值（Outliers）、跳動、階梯狀變化或趨勢[8]。
• 製程能力研究與監控階段：
    ◦ 分析用管制圖 (Analysis Control Charts)：用於回溯評估（Retrospective）。需考慮「錯誤警報（False Alarm）」的機率，只有當超出管制界限的次數多於預期的錯誤警報次數時，才判定製程不穩定[9, 10]。
    ◦ SPC 管制圖 (SPC Control Charts)：用於現場即時控制。任何違反穩定性準則的情況（如超出界限或特定模式）都必須立即採取修正措施[10, 11]。
**5. 統計受控 vs. 受控穩定 (In Statistical Control vs. In Control)**值得注意的是，有些製程雖然不處於嚴格的「統計受控狀態」（例如呈現隨時間變化的分佈模型），但若其變異數值始終落在管制界限內，且遠離規格界限，使得產出不良品的機率極低，這種情況下也可視為「受控（In Control / Controlled Stable）」[12, 13]。這類製程依賴操作員遵循程序進行定期監控與調整，以維持在控制範圍內[13]。

製程性能與能力評估準則
製程性能 (P_p/P_pk) 與製程能力 (C_p/C_pk) 的主要區別在於對**製程穩定性（Stability）**的評估狀態以及其在產品生命週期中的應用階段。
根據來源文件，主要的區別如下：
**1. 穩定性狀態 (Stability Status)**這是兩者最根本的區別。根據 ISO 3534-1 和 ISO 22514 系列標準：
• 製程能力 (C_p,C_pk)：僅在已證實製程滿足穩定性要求（即處於受控狀態，in statistical control）時使用 [1]。如果製程穩定性尚未被證實或處於不穩定狀態，則不能使用此指標 [2]。
• 製程性能 (P_p,P_pk)：用於尚未調查穩定性、數據無法進行穩定性評估，或者已知製程不符合穩定性要求（不穩定/失控）的情況 [1], [2]。如果明確要求提供性能指標，則隱含表示不需要或尚未要求穩定性證明 [3]。
2. 計算方法 (Calculation Method)
• 在數值計算上，這兩類指標（性能與能力）使用相同的公式，皆利用製程的整體變異（overall variation）進行計算 [4], [2]。
• 指標的名稱（P 或 C）是用來標示數據蒐集的邊界條件以及當前的控制狀態 [5]。
3. 應用階段與變異來源 (Application & Variation Sources)
• 初步製程性能 (P_p/P_pk)：通常用於生產製程的放行階段（Release of production processes）。此時通常樣本數較少（例如 n=125），雖然涵蓋了 5M（人、機、料、法、環）的變異來源，但通常僅是有限度的暴露於變異中，且尚未能證實統計上的穩定性 [6], [7], [8]。
• 製程能力 (C_p/C_pk)：用於生產製程的監控階段。此時製程已暴露於完整的變異影響下，且穩定性已被證實 [7], [8]。
**4. 評估矩陣 (Evaluation Logic)**在製程評估的四象限矩陣中：
• 只有同時滿足「穩定」且「有能力」的製程（第一象限），才使用 C_p/C_pk 指標 [9], [10]。
• 處於不穩定狀態的製程（第二、四象限），或穩定性無法驗證時，則使用性能指標進行評估 [9]。
總結來說，若數據來自一個已證明由隨機原因（random causes）主導的穩定製程，應使用 C_p/C_pk；若製程尚未被證明穩定，或數據包含特殊原因（special causes）導致的變異，則應使用 P_p/P_pk [11], [12], [1]。
